# Contributing

Thank you for your interest in contributing to OSM.

## How to contribute

1. Fork the repository and create a feature branch from `master`.
2. Make your changes following the coding guidelines below.
3. Ensure all tests pass (`./devops-stages/stage-test.sh`).
4. Submit a Merge Request against the `master` branch.
5. All discussions in the MR must be resolved before merging.

## Coding guidelines

- Follow [PEP 8](https://peps.python.org/pep-0008/) for Python code.
- Use `black` for code formatting.
- Add Apache 2.0 license headers to all new source files:

  ```
  # Copyright ETSI Contributors and Others.
  # Licensed under the Apache License, Version 2.0 (the "License")
  ```
